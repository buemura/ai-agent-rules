# Rule for API Documentation

## This rule applies to API documentation files in the project.

The api must be documented using OpenAPI specification to ensure clear and consistent communication of the API's structure and functionality. The OpenAPI documentation should include detailed information about the available endpoints, request and response formats, authentication methods, and any other relevant details that developers need to understand how to interact with the API effectively. Additionally, the documentation should be kept up-to-date with any changes made to the API to ensure that it remains accurate and useful for developers.

It should use Swagger latest version to generate interactive API documentation that allows developers to easily explore and test the API endpoints. The documentation should be organized in a clear and logical manner, making it easy for developers to find the information they need quickly. Furthermore, the documentation should include examples of requests and responses for each endpoint to provide practical guidance on how to use the API effectively.

The swagger should be available at the endpoint /docs.
